# Prompt: Sprint Validate

> **Phase 6 — VALIDATE**
> 이 프롬프트는 Claude가 사용자와 함께 실제 사용자 테스트를 설계하고 결과를 분석하는 단계를 실행합니다.
> `02_DEFINE.md`의 Assumption Scorecard와 `06_BUILD.md`의 배포 URL이 필요합니다.
> 결과물은 `docs/sprints/SPRINT-XXX/07_VALIDATE.md`와 `metrics.md`에 기록합니다.

---

## Claude 실행 지침

시작 전 확인:
1. 배포 URL이 작동하는지 확인
2. `02_DEFINE.md`의 Assumption Scorecard 불러오기
3. `02_DEFINE.md`의 Sprint Questions 불러오기
4. 사용자 테스트 참여자 5명이 확보되었는지 확인

---

## STEP 1: 테스트 세션 설계

```
사용자 테스트를 설계합니다.

먼저 테스트 참여자 정보를 알려주세요:
- 몇 명을 인터뷰하나요? (최소 5명 권장)
- 인터뷰 형식은? (대면 / 화상 / 비동기 링크 공유)
- 한 세션에 걸리는 시간은? (권장: 30-45분)

참여자들이 우리 타겟 페르소나와 얼마나 일치하나요?
```

**Claude가 테스트 시나리오 자동 생성** (PRD User Stories 기반):

```markdown
## 사용자 테스트 시나리오

**도입부 스크립트 (5분)**:
"안녕하세요. 오늘은 제가 만든 것을 보여드리는 자리가 아니라,
여러분의 경험을 듣는 자리입니다. 틀린 답은 없습니다.
솔직한 반응이 가장 도움이 됩니다."

---

**Task 1**: [핵심 User Story 기반]
→ 시나리오: "[구체적인 상황 설정]"
→ 관찰 포인트:
  - 어디서 멈추는가?
  - 무엇을 클릭하려 하는가?
  - 어떤 표정/반응을 보이는가?

**Task 2**: [두 번째 User Story 기반]
→ 시나리오: "[구체적인 상황 설정]"
→ 관찰 포인트:
  - [작성]

**마무리 질문 (10분)**:
1. 지금 경험한 것을 한 마디로 표현하면?
2. 이런 제품이 있으면 실제로 사용하시겠어요? (1-10점)
3. 가장 마음에 든 부분은?
4. 가장 불편했던 부분은?
5. 이 문제를 지금 어떻게 해결하고 계신가요?
6. 주변에 비슷한 문제를 겪는 분이 있나요?
```

## STEP 2: 인터뷰 중 실시간 노트 지원

사용자가 인터뷰 도중 또는 직후에 관찰 내용을 공유하면:

```
관찰한 내용을 말씀해주세요.
다음 형식으로 즉시 정리하겠습니다:

- 참여자가 한 행동: [작성]
- 성공 여부: 성공 / 부분 성공 / 실패
- 막힌 지점: [작성]
- 발화한 내용: "[직접 인용]"
- 감정 반응: [작성]
```

**패턴 자동 감지**:
- 3명 이상이 같은 지점에서 막히면 → 즉시 팀에 알림
- 긍정/부정 반응 카운트 실시간 집계
- Assumption과 연결된 관찰은 별도 태깅

---

## STEP 3: 세션 간 패턴 분석

5명의 세션이 끝날 때마다 Claude가 자동 집계:

```
지금까지 [n]명의 테스트가 완료되었습니다.
패턴을 분석합니다.
```

**Claude가 자동 생성하는 패턴 요약**:

```markdown

| [항목 2] | [n]명 | 🟡 Medium |

### 공통 긍정 반응
| 긍정 포인트 | 언급 횟수 |
|-------------|-----------|
| [항목 1] | [n]명 |

### Task 성공률
| Task | 성공 | 부분 성공 | 실패 |
|------|------|-----------|------|
| Task 1 | [n]명 | [n]명 | [n]명 |
| Task 2 | [n]명 | [n]명 | [n]명 |

### 눈에 띄는 발언
- "[직접 인용]" — 참여자 [번호]
- "[직접 인용]" — 참여자 [번호]
```

## STEP 4: Assumption Scorecard 검증

모든 세션 완료 후:


```markdown
[구체적인 테스트 데이터]

```
