## STEP 2: Assumption Scorecard 작성

```
이번 스프린트에서 검증해야 할 핵심 가정들을 정의합시다.

가정(Assumption)이란 "우리가 사실이라고 믿지만 아직 증명되지 않은 것"입니다.

다음 질문으로 가정을 도출합니다:
1. 우리가 틀렸을 때 프로젝트 전체가 흔들리는 가정은 무엇인가요?
2. 지금까지 가장 많이 논쟁이 된 부분은 무엇인가요?
3. 사용자가 "이런 제품이 있으면 쓰겠다"고 했을 때 전제한 것들은 무엇인가요?
```

**Claude가 각 가정을 아래 형식으로 구조화**:

```markdown
### Assumption [번호]

- **가정**: [팀이 사실이라고 믿는 것]
- **왜 믿는가**: [근거 — 인터뷰, 직관, 데이터]
- **틀렸을 때의 결과**: [프로젝트에 미치는 영향]
- **검증 방법**: [Day 5에 어떻게 테스트할 것인가]
- **리스크 등급**: 🔴 High / 🟡 Medium / 🟢 Low
```

**Claude 분석 지침**:

- 리스크가 높은 순서로 정렬
- 5-7개를 초과하면 "가장 중요한 5개만 남겨봅시다"로 우선순위 결정 유도
- 검증 불가능한 가정은 "Day 5에 어떻게 테스트하시겠어요?"로 구체화 요청

## STEP 3: Success Metrics 정의

```
스프린트가 성공했다는 것을 어떻게 알 수 있을까요?

좋은 성공 지표 기준:
✅ Day 5 이후 48시간 안에 측정 가능
✅ 팀 전체가 동의하는 기준
✅ "예/아니오"로 판단 가능
❌ "사용자들이 좋아했다" (측정 불가)
❌ "매출 1억" (5일 안에 측정 불가)

다음을 정의해봅시다:
1. 최소 성공 기준 (이것만 되면 Build 결정): [작성]
2. 이상적인 성공 기준 (이것까지 되면 확신): [작성]
3. 실패 기준 (이것이면 Pivot 또는 Stop): [작성]
```

**Claude 분석 지침**:

- 정량 지표와 정성 지표를 균형 있게 정의
- "5명 중 3명 이상이 다시 쓰겠다고 한다" 형태로 구체화 제안

## STEP 4: MVP Scope 결정

```
이번 스프린트에서 만들 것과 만들지 않을 것을 명확히 구분합시다.

MVP 범위를 결정하는 질문:
1. Day 4 하루 만에 만들 수 있는 것은 무엇인가요?
2. Sprint Question에 답하기 위해 반드시 있어야 하는 기능은?
3. "있으면 좋지만 없어도 테스트 가능한" 기능은?

지금까지 언급된 기능들을 함께 분류해봅시다.
```

**Claude가 즉시 분류 제안**:

```markdown
## MVP Scope 초안

### P0 — 반드시 있어야 함 (Day 4에 구현)

- [기능 A]: [이유 — 어떤 Sprint Question을 검증하는가]
- [기능 B]: [이유]

### P1 — 시간이 남으면 (Day 4 PM 또는 이후)

- [기능 C]: [이유]

### P2 — 다음 스프린트로 (Backlog)

- [기능 D]: [이유]

### Out of Scope (이번에는 절대 안 함)

- [기능 E]: [이유]
```

## STEP 5: Storyboard (6-Frame) 작성

```
우리 솔루션의 경험을 6개 장면으로 그려봅시다.
아직 솔루션을 확정하지 않아도 됩니다. 이상적인 사용자 경험을 상상해보세요.

다음 질문에 답해주세요:
1. 우리 타겟 사용자가 이 문제를 처음 겪는 구체적인 순간은?
2. 가장 고통스러운 "oh crap" 순간은 언제인가요?
3. 우리 제품을 처음 발견하는 현실적인 경로는?
4. 제품을 처음 사용할 때 "아!"하는 순간이 있다면?
5. 제품을 사용한 후 삶이 어떻게 달라지나요?
```

**Claude가 즉시 6-Frame 초안 작성**:

```markdown
### Frame 1: Character & Context

**[구체적 인물명], [역할], [상황]**
→ [현재 상태 묘사]
→ 감정: [감정 상태]

### Frame 2: Problem Emerges

**[마찰 상황]**
→ [어떻게 문제가 드러나는가]
→ 감정: [좌절/불안/혼란]

### Frame 3: "Oh Crap" Moment

**[위기 순간]**
→ [피크 고통의 구체적 상황]
→ 감정: [절박함] → 해결책 탐색 시작

### Frame 4: Solution Appears

**[우리 제품 발견 경로]**
→ [현실적인 발견 방법: 검색/추천/광고]
→ 첫인상: [작성]

### Frame 5: "Aha" Moment

**[핵심 가치 경험 순간]**
→ [무엇이 달라졌는가]
→ 감정: [안도/흥분/만족]

### Frame 6: Life After

**[새로운 일상]**
→ [구체적 변화: 시간/비용/스트레스]
→ "이거 [누구에게] 추천해야겠다"
```
