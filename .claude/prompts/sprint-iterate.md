## 시나리오 A: BUILD (프로덕션 출시)

```
🟢 BUILD 결정이 확인되었습니다.
핵심 가정이 검증되었고, 사용자가 실제 가치를 경험했습니다.

출시를 위한 계획을 함께 수립합니다.
```

**Claude가 GTM 계획 초안 작성**:

```
다음을 결정합시다:

1. 출시 타겟 채널
   - 인터뷰한 사용자들이 주로 어디서 정보를 얻나요?
   - 가장 빠르게 100명을 확보할 수 있는 채널은?

2. 첫 사용자 획득 방법
   - 이미 인터뷰한 5명에게 먼저 초대장 발송
   - 추천 요청 (인터뷰 시 연락처 받은 경우)
   - [확인된 채널] 게시

3. 출시 메시지
   - 사용자 인터뷰에서 가장 많이 나온 언어로 작성
   - "우리는 [문제]를 가진 [사람]을 위해 [솔루션]을 만들었습니다"

4. 30일 목표 지표
   - MAU: [목표]
   - 핵심 액션 완료율: [목표]%
   - NPS: [목표]
```

**출시 체크리스트 자동 생성**:

```markdown
## 출시 전 체크리스트

### 제품

- [ ] 프로덕션 환경 배포 완료
- [ ] 모니터링 설정 (에러 알림)
- [ ] 백업/복구 방법 확인

### 마케팅

- [ ] 랜딩 페이지 최종본 확인
- [ ] 핵심 메시지 1-2개 확정
- [ ] 첫 사용자 초대 이메일/메시지 초안 준비

### 지원

- [ ] 사용자 문의 채널 설정 (이메일 / 채팅)
- [ ] FAQ 초안 작성 (인터뷰에서 나온 질문 기반)

출시 목표 날짜: [작성]
```

## 시나리오 B: ITERATE (개선 후 재검증)

```
🟡 ITERATE 결정이 확인되었습니다.
방향은 맞지만 핵심 경험에서 마찰이 발견되었습니다.

개선 우선순위를 함께 결정합니다.
```

**Claude가 개선 항목 도출**:

```
07_VALIDATE.md의 Pain Point 목록을 기반으로
개선 항목을 우선순위별로 정리합니다.

우선순위 기준:
1. 테스트에서 3명 이상이 언급한 것 (HIGH)
2. Task 성공률을 직접 낮춘 것 (HIGH)
3. 1-2명만 언급했지만 심각한 것 (MEDIUM)
4. 미적/편의성 관련 (LOW)
```

**Claude가 자동 생성하는 개선 계획**:

```markdown

| --- | --------- | ------------------------- | --------- | ------ |

| 3   | [항목]    | [n]명 언급                | [시간]    | [역할] |

### 다음 검증 계획

- 개선 완료 목표: [날짜]
- 재검증 방법: [동일 참여자 / 새 참여자]
- 재검증 날짜: [날짜]
- 성공 기준: [이것이면 BUILD 결정]
```

## STEP 2: Lessons Captured (공통)

모든 시나리오에 해당:

```
시나리오와 관계없이 스프린트 회고를 진행합니다.
솔직하게 답해주세요.
```

**Claude가 회고 퍼실리테이션**:

```
다음 4가지 영역에서 팀의 의견을 들려주세요:

1. KEEP (잘 된 것 — 다음에도 계속)
   무엇이 가장 잘 작동했나요?

2. IMPROVE (아쉬운 것 — 다음엔 더 잘할 수 있는 것)
   어느 단계에서 가장 많은 시간이 낭비됐나요?

3. STOP (하지 말아야 할 것)
   돌이켜보면 하지 말았어야 했던 것은?

4. TRY (시도해볼 것 — 다음 스프린트의 실험)
   다음번에 해보고 싶은 새로운 방법은?
```
