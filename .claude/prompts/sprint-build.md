# Prompt: Sprint Build

> **Phase 5 — BUILD**
> 이 프롬프트는 Claude가 사용자와 함께 Claude Code를 활용한 구현을 관리하는 단계를 실행할 때 사용합니다.
> `04_PRD.md`와 `05_PROMPTS.md`가 완성된 후 실행하세요.
> 진행 상황은 `docs/sprints/SPRINT-XXX/06_BUILD.md`에 실시간 기록합니다.

---

## Claude 실행 지침

시작 전 확인:

1. `04_PRD.md` 완성 확인 (P0 기능, Acceptance Criteria 포함)
2. `05_PROMPTS.md` 완성 확인 (구현 프롬프트 준비됨)
3. 개발 환경 준비 여부 확인
4. Day 5 사용자 테스트 일정 확인 (역산하여 빌드 데드라인 설정)

---

## STEP 1: 빌드 세션 시작

```
BUILD 단계를 시작합니다.

현재 시간을 기준으로 데드라인을 설정합니다:
- 빌드 완료 목표: [날짜/시간]
- 사용자 테스트: [날짜/시간]
- 버퍼: [빌드 완료 ~ 테스트] 사이 최소 12시간 권장

P0 기능 구현 순서를 확인합니다:
[04_PRD.md P0 기능 목록을 종속성 순서대로 재정렬]

1. [기반 기능 먼저] — 다른 기능이 의존하는 것
2. [핵심 기능] — 사용자가 가치를 경험하는 순간
3. [엣지 기능] — 테스트에 필요하지만 보조적인 것

순서가 맞나요? 조정이 필요하면 말씀해주세요.
```

---

## STEP 2: Claude Code 세션 가이드

```
Claude Code 세션을 시작하는 방법을 안내합니다.

Claude Code에 다음을 입력하세요:

---
@file:docs/sprints/SPRINT-001/04_PRD.md
@file:docs/sprints/SPRINT-001/05_PROMPTS.md

위 PRD와 프롬프트를 바탕으로 MVP 구현을 시작합니다.

먼저 Prompt 1 (System Architecture)을 실행하여
프로젝트 구조를 설계한 후, P0 기능을 순서대로 구현해주세요.

구현 규칙:
- 각 기능 완료 후 Acceptance Criteria 체크
- 이슈 발생 시 06_BUILD.md 이슈 로그에 기록
- 기능 추가(scope creep) 금지 — PRD에 없는 것은 구현하지 않음
---
```

**Claude 관리 지침**:

- 빌드 중 Claude Code에서 발생하는 이슈를 사용자가 공유하면 즉시 분석
- "같은 방법으로 재시도"를 막고 대안 접근법 제시
- 30분 이상 막히면 해당 기능을 단순화하거나 스킵하는 방법 제안

---

## STEP 3: 진행 상황 실시간 트래킹

사용자가 진행 상황을 공유할 때마다 `06_BUILD.md`를 업데이트:

```markdown
## 빌드 진행 상황 업데이트

현재 상태를 알려주세요:

- 완료된 기능: [목록]
- 진행 중인 기능: [기능명] — 예상 완료: [시간]
- 막힌 부분: [있다면 설명]

06_BUILD.md를 업데이트하겠습니다.
```

**이슈 발생 시 Claude 분석 프롬프트**:

```
어떤 이슈가 발생했나요?

이슈 유형을 파악합니다:
□ 기술적 버그 (에러 메시지가 있는 경우)
□ 구현 방향 불명확 (PRD가 명확하지 않은 경우)
□ 기술적 한계 (선택한 스택으로 불가능한 경우)
□ 시간 부족 (기능이 예상보다 복잡한 경우)

에러 메시지 또는 문제 상황을 공유해주시면
원인을 분석하고 3가지 해결 방법을 제시합니다.
```

## STEP 4: Scope 조정 의사결정

시간 부족 시 Claude의 트리아지 프로세스:

```
현재 시간: [시간]
사용자 테스트까지: [남은 시간]

남은 P0 기능: [목록]

시간을 고려했을 때 두 가지 선택지가 있습니다:

[옵션 A] 현재 기능들을 완성하고 일부 P0 제외
- 제외 후보: [가장 독립적인 기능]
- 이유: Sprint Question [번호] 검증에 필수가 아님

[옵션 B] 각 기능을 더 단순하게 구현하여 모두 포함
- 단순화 방법: [작성]
- 트레이드오프: [품질 저하 수준]

어떤 옵션을 선택하시겠어요?
결정하면 PRD와 06_BUILD.md를 함께 업데이트합니다.
```

## STEP 5: 배포 및 테스트 준비

```
기능 구현이 완료되면 배포를 진행합니다.

배포 전 체크리스트:
□ 모든 P0 기능이 로컬에서 작동하는가
□ 환경 변수가 올바르게 설정되었는가
□ API 키 등 민감 정보가 코드에 없는가
□ 기본 에러 핸들링이 적용되었는가

05_PROMPTS.md의 Prompt 4 (Deployment)를 Claude Code에서 실행하세요.
배포 완료 후 URL을 공유해주시면 최종 확인을 도와드립니다.
```

**배포 후 스모크 테스트 가이드**:

```markdown
## 배포 후 즉시 확인 사항

다음을 직접 테스트해주세요 (5분):

1. [ ] 랜딩 페이지 접근 가능
2. [ ] 핵심 플로우 처음부터 끝까지 1회 실행
3. [ ] 의도적 에러 입력 시 앱이 다운되지 않음
4. [ ] 모바일 브라우저에서 기본 동작 확인 (해당되는 경우)

이슈 없으면 Day 5 사용자 테스트 준비 완료입니다.
```
