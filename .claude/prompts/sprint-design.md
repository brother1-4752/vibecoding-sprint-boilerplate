## STEP 1: Lightning Demos (경쟁사/참조 분석)

```
솔루션을 스케치하기 전에 이미 존재하는 해결책들을 빠르게 살펴봅시다.
각각 5분 이내로 분석합니다. (이 단계 전체 목표: 30분)

우리가 해결하려는 문제와 비슷한 제품이나 서비스를 알고 계신가요?
직접 경쟁사가 아니어도 됩니다. 참고할 만한 UX 패턴이나 흐름을 가진 것이면 됩니다.

3개를 말씀해주시면 각각 분석하겠습니다.
```

**각 참조 제품에 대해 Claude가 구조화**:

```markdown
### Reference: [제품명]

| 항목                      | 내용                  |
| ------------------------- | --------------------- |
| 어떻게 해결하는가         | [작성]                |
| 우리 타겟 사용자가 쓰는가 | Yes / No / Partially  |
| 배울 점 (Steal)           | [구체적 UX 패턴/기능] |
| 피할 점 (Avoid)           | [약점/갭]             |
| 우리의 차별화 포인트      | [작성]                |
```

**Claude 분석 지침**:

- "우리가 더 잘할 수 있는 것"보다 "우리 사용자가 왜 저걸 쓰는가"에 집중
- 분석 후 경쟁사 공통 강점과 공통 약점을 요약 제시
- 우리의 차별화 포인트 3개를 도출하여 확인받기

## STEP 2: Crazy 8s (아이디어 발산)

```
8가지 다른 방식으로 이 문제를 해결할 수 있다면 어떤 것들이 있을까요?
완성도보다 다양성이 목표입니다. 엉뚱한 아이디어도 환영합니다.

규칙:
- 각 아이디어는 한 문장으로 설명 가능해야 함
- 기술 구현 가능성 판단 금지 (일단 발산)
- 8개를 5분 안에

제가 먼저 8개 초안을 제시하고, 수정 또는 추가해주세요.
```

**Claude가 즉시 Crazy 8s 초안 생성** (문제 맥락 기반):

```markdown
## Crazy 8s — [문제 영역]

1. **[아이디어 1]**: [한 줄 설명] — 핵심 차별화: [작성]
2. **[아이디어 2]**: [한 줄 설명] — 핵심 차별화: [작성]
3. **[아이디어 3]**: [한 줄 설명] — 핵심 차별화: [작성]
4. **[아이디어 4]**: [급진적 변형] — 핵심 차별화: [작성]
5. **[아이디어 5]**: [기존 방식의 반대] — 핵심 차별화: [작성]
6. **[아이디어 6]**: [특정 채널/컨텍스트 특화] — 핵심 차별화: [작성]
7. **[아이디어 7]**: [사용자 행동 기반] — 핵심 차별화: [작성]
8. **[아이디어 8]**: [가장 현실적인 버전] — 핵심 차별화: [작성]
```

## STEP 3: Solution Sketches (3-5개 구체화)

```
Crazy 8s 중 가장 유망한 3-5개를 더 구체적으로 발전시켜봅시다.

어떤 아이디어들이 가장 흥미로웠나요?
또는 제가 가장 가능성 높은 3개를 추천해드릴까요?
```

**Claude가 각 스케치를 구조화**:

```markdown
### Sketch [A/B/C]: [솔루션명]

**핵심 가치 제안**: [한 문장]

**사용자 핵심 플로우**:

1. [진입점] → 사용자가 [행동]
2. [핵심 화면/단계] → [결과]
3. [완료 상태] → [사용자가 얻는 것]

**Sprint Question 연결**:

- 이 솔루션이 답하는 Sprint Question: [번호]

**구현 복잡도**: 🟢 Low / 🟡 Medium / 🔴 High
**사용자 임팩트**: 🟢 Low / 🟡 Medium / 🔴 High

**장점**:

- [작성]

**단점/리스크**:

- [작성]
```

## STEP 4: 솔루션 선택 (Decision)

```
3개의 스케치를 비교해봅시다.

각 스케치를 다음 기준으로 평가하겠습니다:
1. Sprint Question을 얼마나 잘 검증하는가? (1-5점)
2. Day 4 하루 안에 만들 수 있는가? (1-5점)
3. 타겟 사용자의 핵심 Pain Point를 해결하는가? (1-5점)
```

**Claude가 평가 매트릭스 즉시 작성**:

```markdown
| 기준                | Sketch A | Sketch B | Sketch C |
| ------------------- | -------- | -------- | -------- |
| Sprint Q 검증력     | /5       | /5       | /5       |
| 구현 가능성 (Day 4) | /5       | /5       | /5       |
| Pain Point 해결력   | /5       | /5       | /5       |
| **합계**            | **/15**  | **/15**  | **/15**  |
```

```
점수 기준으로 [Sketch X]를 추천합니다.

최종 선택: [Sketch A / B / C]
선택 이유: [작성]

동의하시나요? 다른 의견이 있으시면 말씀해주세요.
```
