# Prompt: Sprint Discover

> **Phase 1 — DISCOVER**
> 이 프롬프트는 Claude가 사용자와 함께 문제를 깊이 이해하는 단계를 실행할 때 사용합니다.
> 결과물은 `docs/sprints/SPRINT-XXX/01_DISCOVER.md`에 기록합니다.

---

## Claude 실행 지침

사용자가 이 단계를 시작하면 Claude는 아래 순서대로 진행합니다.
각 섹션을 순서대로 사용자에게 질문하고, 답변을 받아 `01_DISCOVER.md`에 즉시 정리합니다.
한 번에 모든 질문을 던지지 말고, 대화 흐름에 맞게 진행하세요.

---

## STEP 1: 비즈니스 목표 확정

사용자에게 다음을 질문하세요:

```
이번 스프린트를 통해 해결하고자 하는 문제를 알려주세요.

다음 형식으로 함께 정리해봅시다:
- 우리가 해결하려는 문제는 무엇인가요?
- 이 문제를 해결했을 때 비즈니스적으로 어떤 가치가 생기나요?
- 성공을 어떻게 측정할 건가요? (예: 지표, 수치)
```

**Claude 분석 지침**:
- 답변에서 솔루션이 섞여 있다면 "지금은 문제에만 집중해봐요. 솔루션은 Day 3에 정의합니다"라고 안내
- 목표가 모호하면 "이것이 달성된다면 구체적으로 무엇이 달라지나요?"로 구체화 유도
- 완성 후 한 문장 비즈니스 목표 문장을 제안하고 확인받기

---

## STEP 2: 타겟 사용자 정의

```
이 문제로 가장 고통받는 사람이 누구인가요?

구체적으로 알려주세요:
1. 직업/역할은 무엇인가요?
2. 하루 중 이 문제를 언제 겪나요?
3. 지금 이 문제를 어떻게 해결하고 있나요? (대안 솔루션)
4. 이 문제로 인한 손실은 무엇인가요? (시간/돈/스트레스)
```

**Claude 분석 지침**:
- "모든 사람"이 타겟이라고 하면 가장 고통이 심한 한 그룹으로 좁히도록 안내
- 답변을 바탕으로 프로토-페르소나 초안을 즉시 작성하여 제시
- `user-research/personas.md` 파일에 저장할 것을 안내

---

## STEP 3: 전문가 인터뷰 준비

```
이 문제 영역의 전문가나 현재 사용자와 인터뷰를 진행했거나 진행할 예정인가요?

인터뷰 준비를 도와드리겠습니다. 다음을 알려주세요:
- 인터뷰 대상자의 역할/배경은 무엇인가요?
- 특히 깊이 파고들고 싶은 영역이 있나요?
```

**Claude가 생성할 인터뷰 가이드**:

```markdown
## 인터뷰 가이드 — [대상자 역할]

**목적**: [문제 영역]에서의 실제 경험과 Pain Point 이해

**시간**: 30-45분

### 오프닝 (5분)
- 자기소개와 인터뷰 목적 설명
- "오늘은 제품 소개가 아닌 당신의 경험을 듣는 자리입니다"

### 핵심 질문 (25-35분)
1. [문제 프로세스]에서 가장 힘든 부분이 어디인가요?
2. 지금 이 문제를 어떻게 해결하고 있나요? 도구나 방법이 있나요?
3. 그 방법의 가장 큰 단점은 무엇인가요?
4. 이 문제로 인해 한 달에 얼마나 많은 시간/비용이 낭비되나요?
5. 완벽한 해결책이 있다면 어떤 모습일까요?
6. 이 문제로 어려움을 겪는 동료가 또 있나요?

### 클로징 (5분)
- 가장 중요한 것 하나만 꼽는다면?
- 추가로 인터뷰할 사람 소개 요청 (snowball sampling)
```

## STEP 4: User Journey Map 작성

```
타겟 사용자가 이 문제를 처음 인식하는 순간부터
해결책을 찾고 사용하는 과정을 함께 그려봅시다.

다음 단계별로 답해주세요:
1. 사용자가 이 문제를 처음 인식하는 순간은 언제인가요?
2. 해결책을 찾기 위해 어떤 행동을 하나요?
3. 현재 솔루션을 선택하는 기준은 무엇인가요?
4. 실제 사용 중 어떤 어려움을 겪나요?
```

**Claude 분석 지침**:
- 답변을 받아 아래 형식으로 즉시 User Journey Map 초안 작성

```markdown
| 단계 | 트리거 | 행동 | 감정 | 결과 |
|------|--------|------|------|------|
| Awareness | | | | |
| Consideration | | | | |
| Adoption | | | | |
| Implementation | | | | |
```

- 감정 곡선에서 가장 낮은 지점(최대 고통)을 강조하여 표시
- 이 지점이 우리 제품이 개입해야 할 순간임을 확인
